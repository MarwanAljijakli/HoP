# DeceptionGPT

AI-driven adaptive SSH honeypot. An attacker connects to port `2222`, types any
shell command, and gets a believable response generated by an OpenAI-backed
"cognitive engine" running on `127.0.0.1:5000`. Every interaction is logged to
a JSONL file and surfaced live on a .NET dashboard at
`http://localhost:3002`. If the cognitive engine is unreachable, the honeypot
transparently falls back to hardcoded rule-based responses so the SSH session
never dies, and those fallback events still show up on the dashboard.

## Quick start

See [DeceptionGPT_RUNBOOK.md](DeceptionGPT_RUNBOOK.md) for the full launch
procedure (three detached PowerShell `Start-Process` invocations).

Briefly, you'll need:

- .NET 10 SDK
- Python 3.12+ with `paramiko` installed (`pip install paramiko --break-system-packages`)
- An OpenAI API key exported into the shell where you launch the cognitive
  engine: `$env:OPENAI_API_KEY = "<your-key>"`
- Start, in order: cognitive engine, SSH honeypot, dashboard

The launch commands (PowerShell, all detached via `Start-Process`):

```powershell
# 1. Cognitive engine (port 5000) — needs $env:OPENAI_API_KEY set first
Start-Process python -ArgumentList "-u","app.py" `
  -WorkingDirectory ".\Ai section\Ai section\cognitive" -PassThru -WindowStyle Hidden

# 2. SSH honeypot (port 2222)
Start-Process python -ArgumentList '-u','"Hony bot.py"' `
  -WorkingDirectory ".\DeceptionGPTDashboard\DeceptionGPTDashboard\Views\Home" `
  -PassThru -WindowStyle Hidden

# 3. Dashboard (port 3002)
Start-Process dotnet -ArgumentList "run","--project",".\DeceptionGPTDashboard\DeceptionGPTDashboard","--launch-profile","http" `
  -PassThru -WindowStyle Hidden
```

Then visit <http://localhost:3002> and from another terminal:

```
ssh -p 2222 -o StrictHostKeyChecking=no -o UserKnownHostsFile=NUL anyuser@127.0.0.1
```

(Any password works. Type commands; refresh the dashboard to see them.)

## Security note

This repository is intentionally a research / lab tool. Two things to know:

1. **Bring your own OpenAI key.** No API key is committed to this repo. The
   cognitive engine reads it from `os.environ["OPENAI_API_KEY"]` at startup
   and exits with code 1 if missing. In PowerShell:
   `$env:OPENAI_API_KEY = "<your-key>"` *before* you launch the engine — do
   not paste the key into any file in the project. A real key in a public
   repo gets scraped within minutes.

2. **SSH host key is generated on first run.** `honeypot_host.key` is
   `.gitignore`d on purpose. When you first run the honeypot it generates a
   fresh 2048-bit RSA key alongside the script and reuses it on subsequent
   runs.

Deploy only on isolated lab networks. The dashboard and cognitive engine
bind to `127.0.0.1`; the SSH honeypot binds `0.0.0.0:2222` — adjust your
firewall accordingly.
